# AttriGuard: Defeating Indirect Prompt Injection via Causal Attribution of Tool Invocations

- **Authors**: Yu He, Haozhe Zhu (Zhejiang University), Yiming Li (NTU), Shuo Shao, Zhihao Liu, Zhan Qin (Zhejiang), Hongwei Yao (CityU HK)
- **Arxiv**: https://arxiv.org/abs/2603.10749
- **PDF**: [2603.10749.pdf](2603.10749.pdf)

## Summary

AttriGuard proposes action-level causal attribution — for each proposed tool call, it
verifies necessity by re-executing the agent under a control-attenuated view of external
observations (parallel counterfactual tests). Combines teacher-forced shadow replay,
hierarchical control attenuation, and fuzzy survival criterion robust to LLM stochasticity.

## Key design

- Causal attribution paradigm: "is this tool call causally driven by user intent or by injected content?"
- Parallel counterfactual tests: re-run agent with attenuated external observations
- Teacher-forced shadow replay to prevent attribution confounding
- Fuzzy survival criterion for LLM stochasticity
- 0% ASR under static attacks with negligible utility loss

## Relevance to ActPlane

AttriGuard and ActPlane address overlapping threat models (indirect prompt injection leading
to unauthorized actions) but from different angles:
- AttriGuard: causal attribution at the LLM reasoning layer (is this action causally justified?)
- ActPlane: information-flow enforcement at the kernel layer (has this process touched forbidden data?)
- AttriGuard requires re-execution (expensive); ActPlane's label checks are O(1)
- AttriGuard handles the semantic gap (intent attribution); ActPlane cannot reason about intent
- Complementary: AttriGuard could guard the LLM layer while ActPlane guards the OS layer

# CamQuery: Runtime Analysis of Whole-System Provenance

- **Authors**: Thomas Pasquier (Bristol), Xueyuan Han (Harvard), Thomas Moyer (UNC Charlotte), Adam Bates (UIUC), Olivier Hermant (MINES ParisTech), David Eyers (Otago), Jean Bacon (Cambridge), Margo Seltzer (UBC)
- **Year**: 2018
- **Arxiv**: https://arxiv.org/abs/1808.06049
- **PDF**: [1808.06049.pdf](1808.06049.pdf)

## Summary

CamQuery provides inline, realtime provenance analysis via a Linux Security Module.
It pairs a runtime kernel-level reference monitor (extending CamFlow) with a novel
query module that enables runtime provenance analysis and even mediation of system
events. Modules present a vertex-centric API (like GraphChi/GraphX) — full-graph
analysis routines run as small programs in parallel on every node.

## Key contributions

- Realtime provenance querying (not post-hoc forensics)
- Applications: data loss prevention, intrusion detection, information-flow constraints, regulatory compliance
- Reduces latency of realtime query mechanisms vs prior provenance systems
- Minimal overhead on system execution

## Relevance to ActPlane

CamQuery is a predecessor in the "runtime kernel-level information-flow enforcement"
space. ActPlane's labeled IFC model is conceptually related but differs in:
- Enforcement substrate: eBPF (ActPlane) vs LSM kernel module (CamQuery)
- Policy model: declarative DSL with fixed propagation (ActPlane) vs vertex-centric graph queries (CamQuery)
- Scope: agent-oriented with corrective feedback (ActPlane) vs general system provenance (CamQuery)
- CamQuery builds a full provenance graph; ActPlane uses O(1) label masks without graph storage

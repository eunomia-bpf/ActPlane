# Causality Laundering: Denial-Feedback Leakage in Tool-Calling LLM Agents

- **Authors**: Mohammad Hossein Chinaei (Independent Researcher)
- **Date**: April 2026
- **Arxiv**: https://arxiv.org/abs/2604.04035
- **PDF**: [2604.04035.pdf](2604.04035.pdf)

## Summary

Identifies a novel attack pattern — *causality laundering* — where an adversary probes a
protected action, learns from the denial outcome, and exfiltrates inferred information
through a later seemingly benign tool call. This is not captured by flat provenance tracking
because the leaked information arises from causal influence of the denied action, not direct
data flow.

Presents the Agentic Reference Monitor (ARM), a runtime enforcement layer that consults a
provenance graph over tool calls, returned data, field-level provenance, and denied actions.
ARM propagates trust through an integrity lattice and augments the graph with counterfactual
edges from denied-action nodes.

## Key design

- Causality laundering: denial-feedback leakage via causal (not data) flow
- Provenance graph with counterfactual edges from denied actions
- Integrity lattice for trust propagation
- Field-level provenance tracking
- Sub-millisecond policy evaluation overhead

## Relevance to ActPlane

Directly relevant to ActPlane's corrective-feedback mechanism. ActPlane emits feedback on
denial (EPERM + reason) — this paper shows that denial feedback itself can be an information
channel. Key implications:
- ActPlane's `block` effect returns EPERM — the agent can observe which operations were denied
- Causality laundering could exploit ActPlane's deny-then-feedback pattern
- ARM's counterfactual-edge approach could inform ActPlane's future denial-aware tracking
- ActPlane's label propagation is "flat" (no denied-action edges) — vulnerable to this attack pattern

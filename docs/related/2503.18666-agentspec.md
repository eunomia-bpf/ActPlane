# AgentSpec: Customizable Runtime Enforcement for Safe and Reliable LLM Agents

- **Authors**: Haoyu Wang, Christopher M. Poskitt, Jun Sun (Singapore Management University)
- **Venue**: ICSE 2026, Rio de Janeiro
- **Arxiv**: https://arxiv.org/abs/2503.18666
- **PDF**: [2503.18666.pdf](2503.18666.pdf)

## Summary

AgentSpec is a lightweight domain-specific language for specifying and enforcing
runtime constraints on LLM agents. Users define structured rules incorporating
triggers, predicates, and enforcement mechanisms. Evaluated across code execution,
embodied agents, and autonomous driving.

## Key results

- Prevents unsafe executions in over 90% of code agent cases
- Eliminates all hazardous actions in embodied agent tasks
- 100% compliance for autonomous vehicles
- Computationally lightweight (millisecond overhead)
- Rules can be auto-generated by LLMs (OpenAI o1: 95.56% precision, 70.96% recall)

## Policy model

- Structured rules with triggers + predicates + enforcement actions
- Application-layer enforcement (not kernel-level)
- Modular and interpretable rules

## Relevance to ActPlane

Both are DSL-based runtime enforcement for agents. Key differences:
- Enforcement layer: application-level (AgentSpec) vs kernel-level eBPF (ActPlane)
- Bypass resistance: AgentSpec can be bypassed by direct syscalls; ActPlane enforces below the tool layer
- Information flow: AgentSpec is stateless per-action checks; ActPlane tracks cross-channel label propagation
- AgentSpec's auto-generation of rules from LLMs is complementary — could generate ActPlane DSL policies

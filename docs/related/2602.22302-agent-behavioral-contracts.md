# Agent Behavioral Contracts: Formal Specification and Runtime Enforcement

- **Authors**: Varun Pratap Bhardwaj (Accenture)
- **Date**: February 2026
- **Arxiv**: https://arxiv.org/abs/2602.22302
- **PDF**: [2602.22302.pdf](2602.22302.pdf)

## Summary

Introduces Agent Behavioral Contracts (ABC), a formal framework bringing Design-by-Contract
to autonomous AI agents. A contract C = (P, I, G, R) specifies Preconditions, Invariants,
Governance policies, and Recovery mechanisms as runtime-enforceable components. Defines
(p, δ, k)-satisfaction for probabilistic contract compliance, proves a Drift Bounds Theorem
bounding behavioral drift.

## Key design

- Contract model: Preconditions + Invariants + Governance + Recovery
- (p, δ, k)-satisfaction: probabilistic compliance accounting for LLM non-determinism
- Drift Bounds Theorem: contracts with recovery rate γ > α bound drift to D* = α/γ
- Safe contract composition for multi-agent chains with probabilistic degradation bounds
- Implementation: AgentAssert runtime enforcement library
- Benchmark: AgentContract-Bench (200 scenarios, 7 models, 6 vendors)

## Key results

- Detect 5.2-6.8 soft violations per session that uncontracted agents miss entirely
- 88-100% hard constraint compliance
- Bound behavioral drift to D* < 0.27 across extended sessions
- < 10 ms per action overhead

## Relevance to ActPlane

ABC addresses behavioral drift and probabilistic compliance — a different concern from
ActPlane's deterministic information-flow enforcement. Key differences:
- Focus: behavioral contracts/drift (ABC) vs information-flow control (ActPlane)
- Enforcement: application-level runtime monitor (ABC) vs kernel eBPF (ABC)
- ABC handles LLM non-determinism formally; ActPlane's kernel enforcement is deterministic
- ABC's recovery mechanisms are complementary to ActPlane's corrective feedback

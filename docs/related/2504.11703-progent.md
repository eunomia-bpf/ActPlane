# Progent: Securing AI Agents with Privilege Control

- **Authors**: Tianneng Shi, Jingxuan He, Zhun Wang (UC Berkeley), Hongwei Li, Linyu Wu (UC Santa Barbara), Wenbo Guo, Dawn Song (UC Berkeley)
- **Arxiv**: https://arxiv.org/abs/2504.11703
- **PDF**: [2504.11703.pdf](2504.11703.pdf)

## Summary

Progent secures AI agents via privilege control — symbolic rules over tool names and
arguments that deterministically allow or block tool calls. An LLM auto-generates the
initial policy from the task, and an SMT solver verifies that policy updates are
monotonically confining (can only shrink without approval, preventing silent privilege
escalation).

## Key design

- Security policy: symbolic rules over tool names + typed arguments
- Every tool call checked deterministically against policy
- LLM generates initial policy; SMT solver classifies updates as narrowing (auto-apply) or expansion (require approval)
- Monotonic confinement: action space can only shrink without explicit approval
- Evaluated on AgentDojo and ASB benchmarks

## Relevance to ActPlane

Both enforce deterministic policies independent of model reasoning. Key differences:
- Scope: tool-call layer (Progent) vs syscall layer (ActPlane)
- Policy model: per-tool-call symbolic rules (Progent) vs labeled IFC with cross-channel propagation (ActPlane)
- Progent's SMT-verified monotonic confinement is a novel contribution ActPlane lacks
- Progent cannot track information flow across tool boundaries; ActPlane can
- Bypass resistance: Progent enforces at tool-call API; ActPlane enforces at kernel boundary

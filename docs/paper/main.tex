\begin{abstract}
AI coding agents require harnesses to enforce behavioral contracts
for effective and safe operation: do not
leak secrets, run tests before committing, only write to tmp dir.
Agents operate at three levels:
\emph{intent} (semantic goals and reasoning), \emph{action} (tool calls),
and \emph{behavior} (OS operations: file access, process execution). 
Our empirical study of 64 projects shows that while project harness
instruction-file content is declared in natural language as intent, 63\% of 
consists of behavioral contracts, 80\% of which involve system operations behavior
, and 
78\% of projects require cross-event state tracking.
yet, existing agent harness enforcement leaving an
\emph{intent--behavior gap}:
intent-level prompts are forgettable and probabilistic;
action-level guards check tool calls but miss effects that reach the OS through
subprocesses or information flow.
behavior-level sandboxing is unbypassable but disconnected from intent
making it unsuitable for harness enforcement.

ActPlane provides agent harnesses to bridge the 
intent--behavior gap: agents declare behavioral contracts
as a DSL; ActPlane persist them as OS-level agent harnesses 
enforcement them through in-kernel information-flow analysis and semantic feedback. 
We implement ActPlane with eBPF 
and evaluate on the contracts from the empirical study,
showing that ActPlane can enforce complex contracts with low overhead and
provide actionable feedback to agents.
\end{abstract}

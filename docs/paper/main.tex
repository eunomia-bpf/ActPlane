\begin{abstract}
AI coding agents require harnesses to enforce behavioral contracts: do not
leak secrets, run tests before committing, only write tmp dir.
Agents operate at three levels:
\emph{intent} (semantic goals and reasoning), \emph{action} (tool calls),
and \emph{behavior} (OS operations: file access, process execution). 
Enforcing the behavioral contracts requires coverage span all three levels, from
conversational to system constraints. 
An empirical study of popular projects shows that real contracts
frequently involve cross-object information flow in the OS,
yet, existing agent
harness enforcement operates at a single level, leaving an
\emph{intent--behavior gap}:
intent-level prompts are forgettable and probabilistic;
action-level guards check tool calls but miss effects that reach the OS through
subprocesses or information flow; behavior-level enforcement (syscall
filters, sandboxes) is unbypassable but returns failures disconnected
from intent.

ActPlane provides enforcement for agent harnesses to bridge the 
intent--behavior gap: agents transforming semantic info into
 declarative behavioral 
contracts DSL, ActPlane enforces them as
OS-level information-flow analysis and semantic feedback.
